# 🎟️ Система за управление на събития (Event Management Platform)

Дипломен проект — уеб приложение за публикуване на събития и продажба на билети,
изградено с **ASP.NET Core 8 MVC**.

---

## 📌 Какво прави приложението

- Гостите разглеждат и търсят събития (концерти, спорт, конференции).
- Регистрираните **потребители** купуват билети.
- **Организаторите** създават и управляват свои събития (CRUD).
- **Администраторите** управляват категории и потребители, виждат статистики.
- Има и **REST API** (`/api/events`), което връща данните като JSON.

---

## 🧱 Технологии

| Слой | Технология |
|------|-----------|
| Език / Framework | C# / ASP.NET Core 8 MVC |
| База данни | SQL Server (LocalDB) |
| ORM | Entity Framework Core 8 (Code-First + миграции) |
| Удостоверяване | ASP.NET Core Identity + Роли |
| Тестове | NUnit + Moq + EF Core InMemory |
| Front-end | Razor Views + Bootstrap 5 |

---

## 🏛️ Архитектура (3 слоя)

```
Браузър
   │
   ▼
[ Controllers ]   ← приемат заявки, връщат изгледи/JSON
   │
   ▼
[ Services ]      ← БИЗНЕС ЛОГИКА (правила: места, цени, права)
   │
   ▼
[ DbContext / EF Core ]  ← говори с базата
   │
   ▼
 SQL Server
```

Контролерите **не** пишат заявки към базата директно — те викат сервизите.
Сервизите се подават през **dependency injection** (регистрирани в `Program.cs`).

---

## 🗃️ Модели (entities)

- **ApplicationUser** — потребител (наследява `IdentityUser`).
- **Event** — събитие (заглавие, дата, място, цена, капацитет…).
- **Category** — категория на събитие (Музика, Спорт, Технологии…).
- **Ticket** — билет (връзка между потребител и събитие).
- **Review** — мнение/оценка за събитие (бонус).

## 👤 Роли

## ▶️ Как да стартирам проекта

1. Уверете се, че имате инсталиран **.NET 8 SDK** и **SQL Server LocalDB**.
2. От главната папка изпълнете:

```bash
cd EventManagementSystem
dotnet run
```

3. При първо стартиране базата се създава автоматично (миграции) и се
   попълва с примерни данни (seed).
4. Отворете адреса, който се изписва в конзолата (напр. `https://localhost:7xxx`).

### Команди за базата (по желание)

```bash
dotnet ef migrations add Име      # нова миграция
dotnet ef database update         # прилага миграциите
```

### Стартиране на тестовете

```bash
cd EventManagementSystem.Tests
dotnet test
```
